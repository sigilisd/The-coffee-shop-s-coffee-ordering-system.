# The-coffee-shop-s-coffee-ordering-system.

# Предметная область
Система оформления заказа кофе в кофейне. Пользователь настраивает напиток: база, размер, молоко, сиропы, сахар, лед. Используйте шаблон Builder, что позволяет собирать заказ пошагово, читабельно и с валидацией.

# Ограничения

Язык: Python 3.10+.
Внешние зависимости запрещены (только стандартная библиотека).
Вся реализация в одном файле main.py.
Добавить базовые тесты через assert.

# Что нужно реализовать

1. Модель неизменяемого заказа
Класс CoffeeOrder (value object)
Поля:
  base: одно из ["espresso", "americano", "latte", "cappuccino"].
  size: одно из ["small", "medium", "large"].
  milk: одно из ["none", "whole", "skim", "oat", "soy"], по умолчанию "none".
  syrups: неизменяемая коллекция (tuple[str, ...]), по умолчанию пусто.
  sugar: int, 0 по умолчанию.
  iced: bool, False по умолчанию.
  price: float, вычисляется билдером.
  description: str, человекочитаемое описание заказа.
Требования:
  Метод str возвращает description (или краткую строку с ценой, если description пуст).

1. Fluent Builder для заказа кофе
Класс CoffeeOrderBuilder с внутренним состоянием для всех опций.
Методы текучего интерфейса (возвращают self):
  set_base(base)
  set_size(size)
  set_milk(milk)
  add_syrup(name) — добавляет сироп; игнорирует дубликаты; ограничение на максимум сиропов.
  set_sugar(teaspoons) — валидация по диапазону.
  set_iced(iced: bool = True)
  clear_extras() — сбрасывает добавки (молоко, сиропы, сахар, шоты, лед).
  build() — создает новый CoffeeOrder.
Валидация при build():
  Обязательные поля: base и size — иначе ValueError.
Ограничения и прайсинг (можно вынести в константы билдера):
  Базовые цены:
    espresso: 200
    americano: 250
    latte: 300
    cappuccino: 320
  Множители размера:
    small: 1.0, medium: 1.2, large: 1.4
  Доплаты:
    молоко: whole/skim = 30, oat = 60, soy = 50, none = 0.0
    сироп: 40 за каждый
    лед: 0.2 фиксировано при iced=True
   Лимиты:
    сахар: 0..5
    сиропов: максимум 4
Расчет цены:
   формулу нужно продумать самостоятельно
Описание заказа:
  Строка формата: "<size> <base /> [with <milk> milk] [+syrup list] [(iced)] [<sugar> tsp sugar]"
  Элементы, равные значению по умолчанию, в описание не включать.

1. Чистота и переиспользование билдера
build() возвращает новый CoffeeOrder, не изменяя уже построенные заказы.
Билдер можно переиспользовать для новых заказов, меняя часть опций.
Состояние билдера не «протекает» наружу (например, ни один метод не возвращает внутренние списки).

1. Минимальные тесты
Добавьте в конец файла блок с простыми проверками:

Базовый заказ: задать base, size, несколько опций; проверить:
  правильность типов, непустую цену, наличие опций в объекте заказа.
Переиспользование билдера:
  Собрать order1, затем изменить пару опций и собрать order2; убедиться, что order1 не изменился, а order2 отличается и цена валидна.
Валидация:
  Отсутствует base — build() должен поднять ValueError.
  Отсутствует size — build() должен поднять ValueError.
  Нарушение лимитов (сахар > 5, шотов > 3) — ValueError.
Логические проверки:
  Дубликат сиропа не увеличивает количество сиропов и не влияет на цену второй раз.
  iced добавляет доплату.
  
# Стиль и качество

Аннотации типов обязательны.
Докстринг к CoffeeOrderBuilder с кратким описанием правил и ограничений.
Константы цен и лимитов собрать в одном месте (как атрибуты класса).
Именование методов и переменных — понятное и единообразное.
